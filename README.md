## 📋 Описание

SonicWall SMA Security Scanner - это многопоточный сканер уязвимостей для устройств SonicWall Secure Mobile Access (SMA). Инструмент способен обнаруживать и эксплуатировать различные уязвимости, включая:

- Обход пути (Path Traversal)
- Обход аутентификации
- Внедрение команд
- SQL-инъекции
- Переполнение буфера
- XSS-уязвимости
- Бэкдоры и дефолтные учетные данные

## 🚀 Установка

### Требования

- Go 1.20 или выше
- Git (для установки зависимостей)

### Сборка из исходного кода

```bash
# Клонирование репозитория
git clone https://github.com/1337or420/SonicWall-SMA-Security-Scanner.git
cd sonicwall-scanner

# Инициализация модуля Go
go mod init sonicwall-scanner

# Установка зависимостей
go get github.com/gorilla/websocket

# Сборка
go build -o sonicwall-scanner main.go

# Или сборка с оптимизацией
go build -ldflags="-s -w" -o sonicwall-scanner main.go
```

### Кросс-компиляция

```bash
# Для Linux (amd64)
GOOS=linux GOARCH=amd64 go build -o sonicwall-scanner-linux main.go

# Для Windows (amd64)
GOOS=windows GOARCH=amd64 go build -o sonicwall-scanner.exe main.go

# Для macOS (amd64)
GOOS=darwin GOARCH=amd64 go build -o sonicwall-scanner-mac main.go

# Для macOS (Apple Silicon)
GOOS=darwin GOARCH=arm64 go build -o sonicwall-scanner-mac-arm main.go
```

## 📖 Использование

### Режим Web UI (по умолчанию)

```bash
./sonicwall-scanner
```

После запуска:
1. Автоматически откроется браузер с веб-интерфейсом
2. Web UI будет доступен по адресу: `http://127.0.0.1:{port}`
3. Загрузите список целей или введите их вручную
4. Настройте количество потоков (1-20)
5. Нажмите "START SCAN"

### Режим командной строки (CLI)

```bash
./sonicwall-scanner <targets_file> [threads]
```

**Примеры:**

```bash
# Сканирование с 5 потоками (по умолчанию)
./sonicwall-scanner targets.txt

# Сканирование с 10 потоками
./sonicwall-scanner targets.txt 10

# Сканирование с максимальной производительностью (20 потоков)
./sonicwall-scanner targets.txt 20
```

---

## 📁 Формат файла целей

Файл целей (`targets.txt`) поддерживает несколько форматов (один на строку):

```
# Только IP (порт 443 по умолчанию)
192.168.1.1

# IP с указанием порта
10.0.0.1:8443

# IP, порт и учетные данные
192.168.1.100:4433:admin:password

# Доменное имя
vpn.example.com:443

# Комментарии (начинаются с #)
# Это комментарий
```

## 📊 Результаты

### Файлы результатов

Результаты сохраняются в директории `results/`:

| Файл | Описание |
|------|----------|
| `scan_results.jsonl` | Полные результаты в формате JSON Lines |
| `credentials.txt` | Найденные учетные данные в текстовом формате |

### Структура результатов (JSON)

```json
{
  "target": "192.168.1.1:443",
  "timestamp": "2024-01-15T10:30:00Z",
  "access": "compromised",
  "device_info": {
    "is_sonicwall": true,
    "is_sma": true,
    "model": "SMA 6200",
    "version": "10.2.0.8",
    "port": "443"
  },
  "exploits": [
    {
      "target": "192.168.1.1:443",
      "vulnerable": true,
      "cve": "CVE-2023-44221",
      "name": "Path Traversal to Config Disclosure",
      "details": "File read via: /cgi-bin/sslvpnclient?sw=../../../../etc/passwd",
      "credentials": ["password=SecretPass123"],
      "extracted": "root:x:0:0:root:/root:/bin/bash..."
    }
  ],
  "credentials": ["password=SecretPass123", "admin:sonicwall"]
}
```

### Уровни доступа

| Уровень | Описание |
|--------|----------|
| `none` | Уязвимости не обнаружены |
| `vulnerable` | Обнаружены уязвимости, но нет полного доступа |
| `compromised` | Обнаружены критичные уязвимости или учетные данные |

## 🛠️ Функциональность

### Детектирование устройства

- Автоматическое определение SonicWall устройств
- Определение модели и версии прошивки
- Определение типа устройства (SMA/другое)

### Эксплуатация

1. **Path Traversal (CVE-2023-44221)**
   - Чтение `/etc/passwd`, `/etc/shadow`
   - Извлечение конфигурационных файлов
   - Поиск учетных данных в конфигах

2. **Auth Bypass (CVE-2024-40766)**
   - Извлечение сессионных токенов
   - Обход механизмов аутентификации

3. **Default Credentials**
   - Проверка 14+ стандартных комбинаций
   - Поддержка пользовательских учетных данных

4. **Command Injection (CVE-2021-20039)**
   - Выполнение системных команд
   - Множественные векторы внедрения

5. **Buffer Overflow (CVE-2025-40596)**
   - Проверка на переполнение буфера
   - Детектирование падений сервиса

6. **SQL Injection (CVE-2021-20016)**
   - Union-based инъекции
   - Boolean-based инъекции

7. **XSS (CVE-2025-40598)**
   - Reflected XSS
   - Проверка отражения payload

8. **OVERSTEP Backdoor**
   - Проверка наличия бэкдора
   - Анализ ответов сервера

## 🔧 Конфигурация

### Переменные окружения

```bash
# Включение отладочного режима
export SONICWALL_DEBUG=true

# Установка пользовательского таймаута (секунды)
export SONICWALL_TIMEOUT=30

# Указание прокси
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
```

### Настройка потоков

| Потоки | Использование CPU | Скорость | Рекомендация |
|--------|-------------------|----------|--------------|
| 1-5 | Низкое | Медленная | Для тестирования |
| 5-10 | Среднее | Средняя | Оптимально для большинства |
| 10-20 | Высокое | Быстрая | Для больших списков |

## 📝 Логирование

### Формат логов

```
[2024-01-15 10:30:00] [*] Scanning: 192.168.1.1:443
[2024-01-15 10:30:05] [!!!] COMPROMISED: 192.168.1.1:443
[2024-01-15 10:30:05] [*] Found: CVE-2023-44221 - Path Traversal
[2024-01-15 10:30:05] [*] Credentials: admin:sonicwall
```

### Уровни логирования

- `[*]` - Информация
- `[!]` - Предупреждение/Уязвимость
- `[!!!]` - Критическая находка
